# 香互 · AI 飲食紀錄(階段 1:拍照上傳)

參考 [PICTAMEAL](https://mhealth.jmir.org/2025/1/e60070) 研究中的「AI 影像辨識飲食回報」流程,重製前三格畫面(A 開始 → B 拍照 → C 確認上傳),用手機外框呈現。

**目前狀態**:純前端展示,尚未接 AI 菜色辨識。上傳後的餐點會顯示在下方「上傳紀錄」,狀態標記為「等待辨識」,做為下一階段(逐道菜辨識確認)的資料來源。

## 檔案結構

```
photo-upload-site/
├── index.html   # 手機外框三格畫面 + 下方上傳紀錄
├── style.css    # 香互活力主題樣式(手機外框、畫面切換、標籤)
├── script.js    # 畫面導覽、拍照、選菜色數量、寫入上傳紀錄
└── README.md
```

## 目前的使用者旅程

1. **A 開始**:品牌畫面,按「開始拍照」
2. **B 拍照**:開啟裝置鏡頭,可拍照或改用「從相簿選」上傳既有照片
3. **C 確認上傳**:預覽照片、用圓形標籤選這餐大概幾道菜(1–6+)、按「確認上傳」
4. **D 辨識菜色**:照片上出現每道菜各自的邊界框跟編號圓點,點圓點會跳出提示泡泡告訴你 AI 覺得那是什麼菜,按泡泡上的「✓」確認,圓點會變綠;全部確認完才能按「完成 · 存入日記」
5. 完成的餐點會出現在頁面下方「上傳紀錄」清單,顯示辨識出的菜名清單

**辨識結果目前是假資料**:邊界框位置是用網格 + 隨機微調生成,菜名是從一個固定清單隨機配對,不是真的圖片辨識。要接真的 AI,需要後端服務(例如自建的食物辨識模型,或第三方圖片辨識 API)。

## 本機預覽

```bash
cd photo-upload-site
python3 -m http.server 8000
```
瀏覽器打開 `http://localhost:8000`(拍照功能需要在 `http://localhost` 或 `https://` 下才會拿到鏡頭權限)。
