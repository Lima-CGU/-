# 香互 · AI 飲食紀錄(階段 1:拍照上傳)

參考 [PICTAMEAL](https://mhealth.jmir.org/2025/1/e60070) 研究中的「AI 影像辨識飲食回報」流程,重製前三格畫面(A 開始 → B 拍照 → C 確認上傳),用手機外框呈現。

**目前狀態**:純前端展示。邊界框位置由 OpenCV.js 在瀏覽器裡即時偵測(真實座標),菜名辨識仍是假資料模擬。

## 檔案結構

```
photo-upload-site/
├── index.html   # 手機外框四格畫面 + 下方上傳紀錄,head 內載入 OpenCV.js
├── style.css    # 香互活力主題樣式(手機外框、畫面切換、邊界框/圓點/泡泡)
├── script.js    # 畫面導覽、拍照、OpenCV.js 圓形偵測、辨識確認、寫入上傳紀錄
└── README.md
```

## 目前的使用者旅程

1. **A 開始**:品牌畫面,按「開始拍照」
2. **B 拍照**:開啟裝置鏡頭,可拍照或改用「從相簿選」上傳既有照片
3. **C 確認上傳**:預覽照片、用圓形標籤選這餐大概幾道菜(1–6+)、按「確認上傳」
4. **D 辨識菜色**:上傳確認後,瀏覽器會用 [OpenCV.js](https://docs.opencv.org/) 即時跑 Hough Circle Transform,偵測照片裡圓盤/圓碗的**真實位置**當作邊界框;每個邊界框有一個編號圓點,點下去會跳出提示泡泡,顯示 AI(目前是假資料模擬)覺得那是什麼菜,按「✓」確認後圓點變綠;全部確認完才能按「完成 · 存入日記」
5. 完成的餐點會出現在頁面下方「上傳紀錄」清單,顯示辨識出的菜名清單

**邊界框位置是真的,菜名還是假資料**:
- 位置:瀏覽器載入 OpenCV.js 後,對照片跑圓形邊緣偵測,算出的圓盤/圓碗座標是真實計算出來的,不是隨機排列
- 菜名:目前還是從一個固定清單隨機配對,不是真的食物辨識結果
- 如果照片裡沒有偵測到至少 2 個圓形容器(例如方形餐盒、不規則擺盤,或 OpenCV.js 還沒載入完成),會自動退回原本的網格隨機排列示範版本,並用提示文字告訴使用者

要接上「真的知道這是什麼菜」的功能,還是需要一個食物辨識模型(例如自訓練或第三方 API),這部分尚未實作。

## 本機預覽

```bash
cd photo-upload-site
python3 -m http.server 8000
```
瀏覽器打開 `http://localhost:8000`(拍照功能需要在 `http://localhost` 或 `https://` 下才會拿到鏡頭權限)。

## 下一階段規劃(尚未實作)
